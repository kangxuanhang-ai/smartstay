# C端首页改版设计说明书

## 1. 概述

### 1.1 背景

当前 C端首页视觉效果平淡，缺乏高端酒店应有的震撼感。页面仅包含一个纯色横幅、文字介绍、两个按钮和4个设施卡片，无法有效展示酒店的核心竞争力（AI管家、机器人送物、智能客房等）。

### 1.2 目标

- 打造高端酒店风格的首页，提升视觉冲击力
- 突出展示酒店4大核心亮点
- 区分登录/未登录状态，提供差异化的用户体验
- 引导未登录用户通过功能拦截完成登录

### 1.3 设计原则

- **高端商务风**：渐变色块 + 毛玻璃卡片 + 圆角设计
- **信息层次清晰**：Banner → 用户信息 → 酒店亮点 → 配套设施 → 底部信息
- **登录无感知**：未登录可浏览全部内容，点击核心功能时弹出 LoginBottomSheet

## 2. 页面结构

### 2.1 整体布局

页面采用 `ListView` 纵向滚动，从上到下分为 5 个区块：

| 区块 | 内容 | 登录/未登录 |
|------|------|------------|
| A. Banner | 渐变背景 + 酒店名 + 标语 + 导航/拨号按钮 | 都显示 |
| B. 欢迎卡片 | 用户名 + 房间信息 + 4个快捷入口 | 仅登录后 |
| C. 酒店亮点 | 4个核心功能卡片（2x2网格） | 都显示 |
| D. 配套设施 | 4个设施列表卡片 | 都显示 |
| E. 底部信息 | 酒店地址 + 导航/拨号按钮 | 都显示 |

### 2.2 区块 A：Banner

**布局**：
- 高度：200px
- 背景：线性渐变 `Color(0xFF1A1A2E)` → `Color(0xFF1677FF)`，方向从左上到右下
- 内容垂直居中

**元素**：
- 🏨 emoji，字号 48
- "智宿云大酒店"，白色，字号 24，加粗
- "智慧 · 舒适 · 人文"，白色70%透明度，字号 14
- 两个按钮（水平排列，间距12）：
  - 🗺️ 一键导航（蓝色 `#1677FF`，圆角22）
  - 📞 一键拨号（绿色 `#52C41A`，圆角22）

### 2.3 区块 B：欢迎卡片（仅登录后显示）

**判断条件**：`AuthBloc.state.status == AuthStatus.authenticated`

**布局**：
- 外层容器：白色背景，圆角16，左右margin16，上下margin8
- 内部padding 16

**元素**：
- 第一行："欢迎回来，{userName} 👋"，字号18，加粗
- 第二行："房间 {roomNumber} · {roomType} · {roomStatus}"，字号13，灰色
- 间距16
- 4个圆形快捷入口（水平等分布局）：
  - 🤖 AI管家 → 跳转 `/ai-chat`
  - 💡 控房 → 跳转 `/room-control`
  - 📋 工单 → 跳转 `/work-orders`
  - 📄 账单 → 跳转 `/my`（账单在"我的"页面内）

**快捷入口样式**：
- 圆形容器（56x56），浅蓝色背景 `#F0F5FF`
- emoji居中，字号24
- 下方文字标签，字号11，灰色

**房间信息获取**：
- 调用 `GET /api/rooms/my-room` 获取当前入住房间信息
- 如果未入住（404），显示"暂无入住房间"
- 房型映射：`big_bed` → 大床房，`twin` → 双床房，`suite` → 套房
- 状态映射：`occupied` → 已入住，`vacant` → 空闲

### 2.4 区块 C：酒店亮点

**标题**："✨ 酒店亮点"，字号16，加粗，左右padding16

**布局**：2x2 网格，使用 `GridView.count(crossAxisCount: 2, childAspectRatio: 1.4)`

**4张卡片**：

| 序号 | 图标 | 标题 | 描述 | 跳转 |
|------|------|------|------|------|
| 1 | 🤖 | AI智能管家 | 一句话送水、报修、调温 | `/ai-chat` |
| 2 | 🦾 | 机器人送物 | 自动配送，30分钟送达 | `/work-orders` |
| 3 | 💡 | 智能客房 | 手机控灯光·窗帘·空调 | `/room-control` |
| 4 | 🏊 | 空中花园 | 泳池+健身房+行政酒廊 | `/facility`（健身房） |

**卡片样式**：
- 白色背景，圆角16
- 内部padding 16
- 左上角emoji，字号32
- 标题：字号15，加粗，深灰色
- 描述：字号12，灰色
- 点击时：如果未登录，弹出 LoginBottomSheet；如果已登录，跳转对应页面

**点击拦截逻辑**：
```dart
void _onFeatureTap(String route, {Map<String, dynamic>? extra}) {
  final isLoggedIn = context.read<AuthBloc>().state.status == AuthStatus.authenticated;
  if (!isLoggedIn) {
    LoginBottomSheet.show(context);
    return;
  }
  if (extra != null) {
    context.push(route, extra: extra);
  } else {
    context.go(route);
  }
}
```

### 2.5 区块 D：配套设施

**标题**："🏢 配套设施"，字号16，加粗，左右padding16

**样式改进**（相比当前版本）：
- 每个卡片左侧增加 4px 宽的色条装饰
- 颜色按设施类型区分：健身房蓝色、泳池青色、餐厅橙色、洗衣房紫色
- 保持现有的 ListTile 布局（icon + name + time + price + chevron）

**数据**：保持现有的 `_facilities` 硬编码列表，与后端 `/api/hotel/facilities` 对齐。

### 2.6 区块 E：底部信息

**布局**：
- 浅灰色背景 `#F8F9FA`
- 内部padding 20

**元素**：
- 📍 酒店地址：北京市朝阳区建国路100号，字号13，灰色
- 间距12
- 两个按钮（水平排列，与Banner区按钮样式一致）

## 3. 交互逻辑

### 3.1 登录状态判断

```dart
final isLoggedIn = context.watch<AuthBloc>().state.status == AuthStatus.authenticated;
```

### 3.2 未登录用户

- 可以看到所有区块（A、C、D、E）
- 区块B（欢迎卡片）不显示
- 点击酒店亮点卡片 → 弹出 LoginBottomSheet
- 点击设施卡片 → 正常跳转（设施详情页不需要登录）
- 导航/拨号按钮 → 正常工作

### 3.3 已登录用户

- 显示所有区块（A、B、C、D、E）
- 点击酒店亮点卡片 → 直接跳转对应页面
- 欢迎卡片的快捷入口 → 跳转对应页面

### 3.4 房间信息加载

- 在 `initState` 中调用 `GET /api/rooms/my-room`
- 加载中：欢迎卡片显示 skeleton 占位
- 加载成功：显示房间号、房型、状态
- 加载失败（404）：显示"暂无入住房间"
- 加载失败（其他）：显示"信息加载失败"+ 重试按钮

## 7. 不做的事

- 不做图片轮播（用户选择了渐变+标语）
- 不做底部常驻登录入口（用户选择了功能拦截登录）
- 不从后端动态加载酒店信息和设施列表（当前硬编码足够，后续可改）
- 不做动画/动效（保持简洁，后续可加）

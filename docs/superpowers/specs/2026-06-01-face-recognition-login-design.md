# 人脸识别登录功能设计说明书

## 1. 概述

### 1.1 背景

当前 SmartStay C 端登录需要输入身份证号和密码，体验不够便捷。住客入住后，希望能在 C 端通过刷脸直接登录，无需记忆密码。

### 1.2 目标

- B 端开房时录入住客人脸，且人脸必须与身份证匹配
- C 端支持刷脸登录，配合活体检测防止照片攻击
- 登录成功后自动进入对应住客账号

### 1.3 设计原则

- **安全性**：活体检测防照片/视频攻击，人脸与身份证双重验证
- **便捷性**：C 端刷脸即登录，无需输入任何信息
- **轻量化**：后端不加载深度学习模型，全部通过阿里云 API 实现

## 2. 技术架构

### 2.1 使用的阿里云 API

| API | 所属服务 | 用途 |
|-----|---------|------|
| `RecognizeIdcard` | 人证照识别 | OCR 扫描身份证，提取姓名、身份证号、证件照 |
| `DetectFace` | 人体识别 | 检测人脸质量（角度、清晰度、是否闭眼） |
| `CompareFace` | 人体识别 | 证件照 vs 本人脸，1:1 比对验证 |
| `DetectLivingFace` | 人体识别 | C 端活体检测，防照片/视频攻击 |
| `CreateFaceDb` | 人体识别 | 创建人脸库（一次性操作） |
| `AddFace` | 人体识别 | B 端开房时将人脸添加到人脸库，绑定 guest_id |
| `SearchFace` | 人体识别 | C 端登录时从人脸库中 1:N 搜索匹配人脸 |

### 2.2 整体架构

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   B 端 React  │────→│  FastAPI 后端  │────→│   阿里云 API   │
│  (前台开房)    │     │              │     │              │
└──────────────┘     └──────┬───────┘     └──────────────┘
                            │
                            │ PostgreSQL
                            ▼
                     ┌──────────────┐
                     │   数据库      │
                     │ guests 表    │
                     │ (含 face_id) │
                     └──────────────┘
                            │
┌──────────────┐            │
│  C 端 Flutter │────────────┘
│  (住客刷脸)   │────→ 阿里云 API（活体检测 + 人脸搜索）
└──────────────┘
```

### 2.3 人脸库说明

使用阿里云人脸搜索 API 管理人脸库：
- 通过 `CreateFaceDb` 创建人脸库，名称：`smartstay_faces`
- B 端开房时调用 `AddFace` 添加人脸，`EntityId` 绑定 `guest_id`
- C 端登录时调用 `SearchFace` 从库中搜索匹配，返回 `EntityId`（即 `guest_id`）

## 3. B 端开房流程（人脸录入）

### 3.1 流程概述

前台在 B 端为住客办理入住时，完成人脸录入和身份验证。

### 3.2 详细步骤

```
步骤1：前台输入住客身份证号
  ↓
步骤2：获取身份证照片（二选一）
  • 方式A：OCR 扫描 — 笔记本摄像头拍摄身份证正面，调用 RecognizeIdcard 提取信息和证件照
  • 方式B：手动上传 — 上传一张身份证照片文件
  ↓
步骤3：摄像头拍摄住客本人 face
  ↓
步骤4：调用 DetectFace 检测人脸质量
  • 检查照片中是否有人脸
  • 检查人脸角度是否过偏
  • 检查是否闭眼
  • 不合格 → 提示重新拍摄
  ↓
步骤5：调用 CompareFace 比对（证件照 vs 本人脸）
  • 返回相似度分数
  • 相似度 ≥ 阈值（如 80%）→ 匹配成功
  • 相似度 < 阈值 → 拒绝，提示"人脸与身份证不匹配"
  ↓
步骤6：匹配成功后，调用 AddFace 添加人脸到阿里云人脸库
  • 参数：DbName="smartstay_faces"，EntityId=guest_id，ImageBase64=本人脸照片
  • 阿里云自动提取特征并存储
  ↓
步骤7：完成开房，住客获得房间
```

### 3.3 B 端 UI 变更

在现有的开房流程中增加人脸录入步骤：

| 位置 | 变更 |
|------|------|
| 开房弹窗 | 新增"人脸录入"步骤，显示摄像头预览 |
| 身份证输入后 | 显示 OCR 识别结果（姓名、身份证号）供确认 |
| 拍照按钮 | "拍摄住客 face"按钮，点击后拍照 |
| 比对结果 | 显示"匹配成功 ✓"或"匹配失败 ✗" |
| 开房按钮 | 人脸匹配成功后才可点击 |

### 3.4 异常处理

| 异常情况 | 处理方式 |
|----------|---------|
| 摄像头未授权 | 提示"请允许浏览器使用摄像头" |
| 人脸检测失败（无人脸/角度差） | 提示"请重新拍摄，确保 face 正对摄像头" |
| 人脸与身份证不匹配 | 拒绝开房，提示"人脸与身份证信息不匹配" |
| 阿里云 API 调用失败 | 提示"服务暂时不可用，请稍后重试" |
| OCR 识别失败 | 提示"身份证识别失败，请手动上传照片" |

## 4. C 端登录流程（刷脸登录）

### 4.1 流程概述

住客在 C 端 Flutter App 上通过刷脸完成登录，无需输入身份证号和密码。

### 4.2 详细步骤

```
步骤1：住客在登录页点击"刷脸登录"按钮
  ↓
步骤2：打开前置摄像头，显示人脸检测框
  ↓
步骤3：住客将 face 对准检测框，系统自动拍照
  ↓
步骤4：调用 DetectLivingFace（活体检测）
  • 确认是真人，不是照片/视频
  • 非真人 → 提示"检测到非活体，请本人 face 对准摄像头"
  ↓
步骤5：活体检测通过后，调用 SearchFace（人脸搜索）
  • 参数：DbName="smartstay_faces"，ImageBase64=刷脸照片
  • 返回匹配的 EntityId（即 guest_id）和相似度分数
  ↓
步骤6：相似度 ≥ 阈值（如 85%）
  • 找到住客 → 签发 JWT → 登录成功 → 进入首页
  ↓
步骤7：相似度 < 阈值或未找到
  • 提示"未找到匹配的人脸，请先到前台登记入住"
```

### 4.3 C 端 UI 变更

在登录页新增刷脸登录入口：

| 位置 | 变更 |
|------|------|
| 登录页 | 新增"刷脸登录"按钮（在身份证号/密码表单下方） |
| 刷脸页面 | 全屏摄像头预览 + 人脸检测框 + 提示文字 |
| 活体检测中 | 显示"正在验证..."加载状态 |
| 登录成功 | 自动跳转首页 |
| 登录失败 | 显示失败原因 + "重试"和"返回密码登录"按钮 |

### 4.4 异常处理

| 异常情况 | 处理方式 |
|----------|---------|
| 摄像头未授权 | 提示"请允许 App 使用摄像头" |
| 活体检测失败 | 提示"检测到非活体，请本人 face 对准摄像头" |
| 人脸未注册 | 提示"未找到匹配人脸，请先到前台登记入住" |
| 相似度不够 | 提示"识别失败，请重试或使用密码登录" |
| 阿里云 API 调用失败 | 提示"服务暂时不可用，请使用密码登录" |

## 5. 数据库变更

### 5.1 Guest 表新增字段

```sql
ALTER TABLE guests ADD COLUMN face_id VARCHAR(64);  -- 阿里云人脸库中的人脸 ID
ALTER TABLE guests ADD COLUMN face_registered BOOLEAN DEFAULT FALSE;  -- 是否已录入人脸
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `face_id` | VARCHAR(64) | 阿里云人脸库中的人脸唯一标识，录入时由 API 返回 |
| `face_registered` | BOOLEAN | 是否已完成人脸录入，默认 FALSE |

### 5.2 不需要新建表

人脸数据存储在阿里云云端人脸库中，本地数据库只存 `face_id` 作为关联键。

## 6. 后端 API 接口

### 6.1 新增接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/face/ocr-idcard` | POST | OCR 扫描身份证，返回姓名、身份证号、证件照 |
| `/api/face/verify` | POST | 人脸比对（证件照 vs 本人脸），返回是否匹配 |
| `/api/face/register` | POST | 将人脸添加到阿里云人脸库，绑定 guest_id |
| `/api/face/search` | POST | C 端刷脸登录，活体检测 + 人脸搜索 |

### 6.2 接口详情

#### POST `/api/face/ocr-idcard`

**请求**：multipart/form-data，包含身份证照片文件

**响应**：
```json
{
  "name": "张三",
  "id_card_number": "110101199001011234",
  "face_image_url": "base64 编码的证件照"
}
```

#### POST `/api/face/verify`

**请求**：
```json
{
  "id_card_face_image": "base64 证件照",
  "live_face_image": "base64 本人脸照片"
}
```

**响应**：
```json
{
  "matched": true,
  "confidence": 92.5
}
```

#### POST `/api/face/register`

**请求**：
```json
{
  "guest_id": "uuid",
  "face_image": "base64 本人脸照片"
}
```

**响应**：
```json
{
  "success": true,
  "face_id": "阿里云返回的人脸 ID"
}
```

#### POST `/api/face/search`

**请求**：multipart/form-data，包含人脸照片文件

**响应**：
```json
{
  "success": true,
  "guest_id": "uuid",
  "confidence": 95.2,
  "token": "JWT 登录令牌"
}
```

## 8. 安全考虑

### 8.1 防攻击措施

| 攻击方式 | 防御手段 |
|----------|---------|
| 照片冒充登录 | DetectLivingFace 活体检测 |
| 视频冒充登录 | DetectLivingFace 静默活体检测 |
| 人脸与身份证不匹配 | CompareFace 1:1 比对验证 |
| API 滥用 | 后端限流，每次登录最多 3 次尝试 |

### 8.2 隐私保护

- 人脸照片不存储在本地服务器，仅存储在阿里云人脸库
- 本地数据库只存 `face_id`（一串无意义的标识符）
- 住客退房后可选择删除人脸数据

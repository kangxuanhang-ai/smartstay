# C 端语音输入功能设计

## 概述

为 C 端 AI 聊天页面新增语音输入功能，住客可以点击麦克风按钮录音，录音结束后通过阿里云 ASR 识别为文字，填入输入框供住客确认后发送。

## 目标

- 住客可以通过语音代替打字与 AI 对话
- 识别结果填入输入框（不自动发送），住客可编辑后确认
- 录音最长 60 秒，录音中显示计时器和脉冲动画
- 复用现有阿里云凭据，不引入新的第三方服务

## 非目标

- 实时流式语音识别（方案 B，后续可升级）
- 语音合成（AI 回复语音）
- 图片识别（独立功能，后续实现）

## 交互流程

```
点击麦克风 → 开始录音（红色脉冲 + 计时器）
    ↓
再次点击麦克风（或达到 60 秒）→ 停止录音
    ↓
上传音频到 POST /api/ai/transcribe → 显示"识别中..."
    ↓
识别完成 → 文字填入输入框
    ↓
住客编辑/确认 → 点击发送
```

## 前端设计

### 状态管理（ChatBloc）

新增事件：
- `ChatVoiceRecordStarted` — 开始录音，UI 切换到录音状态
- `ChatVoiceRecordStopped` — 停止录音，触发上传和识别
- `ChatVoiceTranscribed` — 识别完成，将文字填入输入框

新增状态字段（ChatState）：
- `isRecording: bool` — 是否正在录音
- `isTranscribing: bool` — 是否正在识别
- `recordingDuration: int` — 录音时长（秒）

### UI 变化

**麦克风按钮（现有位置）：**
- 默认状态：灰色麦克风图标（现有）
- 录音中：红色麦克风图标 + 脉冲动画
- 识别中：灰色麦克风图标 + 加载指示器

**输入栏变化：**
- 录音中：发送按钮变为红色停止按钮，文本输入框显示"录音中 00:05..."
- 识别中：发送按钮变为加载指示器，文本输入框显示"识别中..."

**计时器：**
- 录音开始后显示秒数递增
- 达到 60 秒自动停止录音

### 录音实现

使用 `record` 包（已安装）：
- 音频格式：AAC (m4a)
- 采样率：16kHz（阿里云 ASR 推荐）
- 声道：单声道
- 最大时长：60 秒

权限处理：
- 首次使用弹出麦克风权限请求
- 权限被拒绝时显示引导文字："请在设置中开启麦克风权限"
- 使用 `permission_handler` 包（已安装）

错误处理：
- 录音 < 1 秒：提示"录音时间太短"
- 录音失败：提示"录音失败，请重试"
- 网络错误：提示"识别失败，请重试"
- ASR 返回空结果：提示"未识别到语音内容"

### 识别后行为

识别完成后的文字填入 TextField 输入框，不自动发送。住客可以：
1. 直接点击发送
2. 编辑文字后发送
3. 清除后重新录音

## 后端设计

### 新增端点

```
POST /api/ai/transcribe
Content-Type: multipart/form-data

请求参数：
  - audio: UploadFile (m4a/wav/mp3, 最大 10MB)

响应：
  200: {"text": "识别出的文字内容"}
  400: {"detail": "错误原因"}
  500: {"detail": "ASR 服务调用失败"}

认证：需要 JWT（复用 get_current_guest 依赖）
```

### 阿里云 ASR 集成

使用阿里云"一句话识别"REST API：
- 服务：智能语音交互（NLS）
- 接口：`nls-filetrans`（文件转写）
- 语言：中文普通话
- 采样率：16kHz

实现步骤：
1. 接收上传的音频文件
2. 将音频文件读入内存
3. 调用阿里云一句话识别 API，直接传入音频数据
4. 返回识别文字

依赖：`alibabacloud-nls` Python SDK

### 配置项

复用现有阿里云凭据：
- `ALIYUN_ACCESS_KEY_ID`
- `ALIYUN_ACCESS_KEY_SECRET`
- `ALIYUN_REGION_ID`（默认 cn-shanghai）

需要新增：
- `ALIYUN_ASR_APP_KEY` — 阿里云智能语音交互项目 App Key

### 安全约束

- 端点需要 JWT 认证
- 音频文件大小限制 10MB
- 录音时长限制 60 秒（前端 + 后端双重校验）
- 音频数据仅在内存中处理，不落盘，识别完成后丢弃

## 阿里云控制台前置条件

使用前需要在阿里云控制台完成：
1. 开通"智能语音交互"服务
2. 创建项目（获取 App Key）
3. 配置 RAM 权限（NLS 访问权限）

## 测试计划

### 手动测试

1. 首次使用：弹出麦克风权限请求
2. 权限拒绝：显示引导文字
3. 正常录音 → 停止 → 识别 → 文字填入输入框
4. 录音 < 1 秒：提示"录音时间太短"
5. 录音达到 60 秒：自动停止
6. 网络断开时录音：提示"识别失败，请重试"
7. 编辑识别结果后发送：AI 正常回复
8. 录音中点击发送按钮：变为停止按钮

### 后端测试

1. 正常音频文件：返回识别文字
2. 无音频文件：返回 400
3. 超大文件（>10MB）：返回 400
4. 无 JWT：返回 401
5. ASR 服务异常：返回 500

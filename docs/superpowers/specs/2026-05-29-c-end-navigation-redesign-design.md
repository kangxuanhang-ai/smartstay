# C端导航重构设计规格

> 日期: 2026-05-29
> 状态: 待审核
> 范围: Flutter C端 App 导航结构、登录流程、"我的"页面重构

---

## 1. 背景与目标

当前 C 端 App 的导航逻辑不符合主流酒店 App 的使用习惯：
- 启动后需要先登录才能使用，首页虽然是公开的但 Tab 跳转会强制拦截到登录页
- "账单"Tab 功能单薄，缺乏个人信息管理能力
- 登录流程是独立页面跳转，体验割裂

**目标：** 学习华住会、万豪等主流酒店 App 的设计，实现"先浏览后登录"的体验，让用户在未登录时也能浏览 App 内容，通过友好的引导促使登录。

---

## 2. 导航结构

### 2.1 底部 Tab 栏

| 序号 | Tab 名称 | 图标 | 路由 | 说明 |
|------|----------|------|------|------|
| 1 | 首页 | 🏠 | `/home` | 公开浏览，不变 |
| 2 | 控房 | 💡 | `/room-control` | 智能设备控制（需入住） |
| 3 | 管家 | 🤖 | `/ai-chat` | AI 虚拟管家对话（需登录+入住） |
| 4 | 服务 | 🔧 | `/work-orders` | 服务工单进度（需登录） |
| 5 | 我的 | 👤 | `/my` | 个人信息+订单+账单+设置（原"账单"Tab） |

### 2.2 关键变化

- **"账单"Tab → "我的"Tab**：账单内容融合到"我的"页面中，同时增加个人信息、订单记录、设置模块
- **所有 Tab 永远可点击**：不会灰掉或禁用，未登录时各页面自行处理引导
- **登录改为底部弹窗**：不再跳转独立登录页，保持用户上下文
- **登录后留在当前页**：关闭弹窗，当前页自动刷新为已登录内容

## 3. 未登录状态各页面行为

### 3.1 首页（无变化）

- 正常展示酒店信息、设施、房型等公开内容
- 无需任何登录引导

### 3.2 控房页

- **展示方式：** 控件（灯开关、窗帘滑条、空调温度/模式）以**置灰（opacity 0.25）**状态展示，不可操作
- **引导方式：** 页面底部浮出白色引导条，显示文字"入住后即可控制房间设备" + "立即登录"按钮
- **点击登录按钮 →** 弹出底部登录弹窗

### 3.3 管家页

- **展示方式：** 全页遮罩，居中显示 AI 管家图标（🤖）+ 标题"AI 虚拟管家" + 描述"登录后即可享受智能对话，控制设备、查询信息、提交服务" + "立即登录"按钮
- **背景：** 半透明毛玻璃效果（`rgba(248,249,252,0.95)` + `backdrop-filter: blur(8px)`）

### 3.4 服务页

- **展示方式：** 全页遮罩，居中显示服务图标（🔧）+ 标题"服务工单" + 描述"登录后查看工单进度，提交维修、保洁、送物等服务" + "立即登录"按钮
- **背景：** 与管家页风格统一

### 3.5 我的页（详见第5节）

- 未登录时显示渐变登录卡片 + 设置菜单

## 4. 底部弹窗登录

### 4.1 触发方式

在任何未登录页面点击"立即登录"按钮时，从底部滑出半屏弹窗。

### 4.2 弹窗布局

```
┌─────────────────────────────┐
│          ─── (拖拽条)        │
│                    ✕ (关闭)  │
│                             │
│            🏨               │
│        登录智宿云            │
│     享受智慧入住体验          │
│                             │
│  ┌───────────────────────┐  │
│  │ 身份证号               │  │
│  │ 13042920030603401X    │  │
│  └───────────────────────┘  │
│  ┌───────────────────────┐  │
│  │ 密码          (蓝框)   │  │
│  │ ••••••••               │  │
│  └───────────────────────┘  │
│                             │
│  [      立即登录      ]     │
│                             │
│  还没有账号？请前往前台办理入住 │
└─────────────────────────────┘
```

### 4.5 交互细节

- **背景遮罩：** 毛玻璃效果，点击遮罩可关闭弹窗
- **关闭按钮：** 右上角圆形 ✕ 按钮
- **表单字段：** 身份证号（18位）+ 密码
- **输入框样式：** 默认 `#f5f7fa` 背景，聚焦时蓝色边框 + 浅蓝背景
- **登录按钮：** 渐变蓝色圆角按钮，全宽

### 4.4 错误处理

- 登录失败：弹窗内显示红色错误提示文字，弹窗不关闭
- 网络异常：显示"网络异常，请重试"

### 4.5 首次登录改密流程

当后端返回 `is_first_login: true` 时：

1. 登录成功后，弹窗内容**无缝切换**为"修改初始密码"表单
2. 表单字段：原密码、新密码、确认新密码
3. 改密成功后：关闭弹窗，页面刷新为已登录内容
4. 改密失败：弹窗内显示错误提示，不关闭

### 4.6 登录成功后行为

- 关闭底部弹窗
- **留在当前页面**（不跳转）
- 当前页面自动刷新为已登录内容（重新请求 API）
- AuthBloc 状态更新为 `authenticated`

**模块说明：**

| 模块 | 内容 | 点击行为 |
|------|------|----------|

- 个人信息卡不显示"当前入住"行
- "我的订单"只显示历史记录
- "账单消费"显示"暂无账单"

## 6. 路由变更

### 6.1 新增路由

| 路由 | 页面 | 说明 |
|------|------|------|
| `/my` | MyPage | 新的"我的"页面 |
| `/my/orders` | OrderListPage | 订单列表二级页 |
| `/my/bill` | BillDetailPage | 账单详情二级页（复用现有 BillPage 逻辑） |

### 6.2 移除/调整路由

| 原路由 | 变化 |
|--------|------|
| `/bill` | 移除，账单功能合并到 `/my` |
| `/login` | 保留但不再作为主要登录入口，仅用于深度链接 |

### 6.3 路由守卫变更

- **移除强制登录拦截**：所有路由不再因未登录而跳转到 `/login`
- **匿名白名单扩大**：所有路由都允许匿名访问
- **页面级权限控制**：各页面自行判断登录状态，展示对应 UI

## 7. 技术实现要点

### 7.1 新增组件

| 组件 | 说明 |
|------|------|
| `LoginBottomSheet` | 底部登录弹窗，支持身份证+密码登录，首次登录自动切换改密流程 |
| `AuthGate` | 通用的未登录状态包装器，根据页面类型显示不同的引导 UI |
| `MyPage` | "我的"页面主组件 |
| `OrderListPage` | 订单列表二级页 |
| `BillDetailPage` | 账单详情二级页（迁移自现有 BillPage） |

### 7.2 修改组件

| 组件 | 变化 |
|------|------|
| `BottomNav` | 5 个 Tab：首页、控房、管家、服务、我的（原"账单"改为"我的"） |
| `AppRouter` | 新增路由，移除强制登录拦截 |
| `RoomControlPage` | 未登录时控件置灰 + 底部引导条 |
| `AIChatPage` | 未登录时显示遮罩引导 |
| `WorkOrderPage` | 未登录时显示遮罩引导 |
| `AuthBloc` | 登录成功后通知页面刷新 |

### 7.3 状态管理

- `AuthBloc` 不变，负责登录/登出/改密/用户信息
- 各页面在 `initState` 中检查 `AuthBloc.state.status`，决定显示哪种 UI
- 登录成功后通过 `BlocListener` 监听状态变化，触发页面数据刷新
